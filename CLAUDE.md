## Project Overview

**Identity** is a GTK4/libadwaita GNOME application (written in Rust) for comparing multiple versions of an image or video side by side. It supports tabbed, row, and column display modes and uses GStreamer for video playback and glycin for image rendering.

- License: GPL-3.0+
- App ID: `org.gnome.gitlab.YaLTeR.Identity`
- Upstream: https://gitlab.gnome.org/YaLTeR/identity

## Build System

The project uses **Meson + Cargo** (not plain Cargo). Do not build with `cargo build` directly — use Meson.

### Development build

```bash
meson -Dprofile=development -Dprefix=$PWD/install _build
ninja -C _build install
```

The `development` profile builds in debug mode and sets up the git pre-commit hook for `rustfmt`.

### Release build

```bash
meson -Dprofile=default -Dprefix=/usr _build
ninja -C _build install
```

### Run after build

```bash
ninja -C _build install && MESON_DEVENV=1 ./_build/src/identity
```

The `MESON_DEVENV=1` env var tells the app to load `resources.gresource` from the build directory instead of the install prefix.

### Formatting (enforced by pre-commit hook)

```bash
cargo fmt --all
```

Check only (what the pre-commit hook runs):

```bash
cargo fmt --all -- --check
```

### Run tests

```bash
cargo test
```

Unit tests exist in `src/window.rs` (scale parsing) and potentially other modules. There are no integration tests.

## Architecture

All application logic lives in `src/`. UI definitions are in `data/resources/ui/` (`.ui` files compiled into a GResource bundle).

### Key modules

| File | Role |
|------|------|
| `main.rs` | Entry point: GStreamer init, tracing setup, GSK renderer selection |
| `application.rs` | `GtkApplication` subclass, window management, VA display state |
| `window.rs` | Main window: display modes (tabbed/row/column), player controls, zoom, rotation |
| `page.rs` | One open file — wraps either a GStreamer `playbin3` (video) or a glycin texture (image) |
| `picture.rs` | Custom `GtkWidget` that renders a `GdkPaintable` with zoom, scroll, and rotation |
| `player.rs` | Coordinates GStreamer playback across all open video pages simultaneously |
| `page_grid.rs` | Grid container used in row/column display modes |
| `texture_paintable.rs` | `GdkPaintable` wrapping a `GdkTexture` (static images) |
| `thumbnail_paintable.rs` | `GdkPaintable` for image loading/thumbnail display |
| `glycin.rs` | Integration with the glycin image loading library |
| `scale_request.rs` | Enum: `FitToAllocation` or `Set(f64)` — the zoom state |
| `utils.rs` | Helpers: fractional scale, shortcut construction |
| `config.rs` | Generated by Meson from `config.rs.in` — app ID, version, paths |

### GObject pattern

All custom widgets use the standard gtk-rs GObject subclassing pattern:
- `mod imp { ... }` contains the implementation struct with `ObjectSubclass`
- `glib::wrapper! { ... }` at the bottom exposes the public type
- Properties are declared with `#[property(...)]` macros and `#[derive(Properties)]`
- Template callbacks via `#[template_callback]` and `#[gtk::template_callbacks]`

### Display modes

`Window` supports three display modes stored as `DisplayMode` enum:
- **Tabbed** — uses `adw::TabView`, pages wrapped in `adw::Bin`
- **Row** — uses `PageGrid` with horizontal `gtk::Orientation`
- **Column** — uses `PageGrid` with vertical `gtk::Orientation`

Switching modes transfers live `Page` objects between the two containers without recreating them.

### Video playback

`Player` holds a pool of GStreamer `playbin3` elements. All open video pages are synchronized: play/pause, seek, and playback rate apply to all simultaneously. The `Page` attaches/detaches its `playbin3` to the shared `Player`.

### Image rendering

Images are loaded asynchronously via glycin. The result is a `GdkTexture` exposed through `TexturePaintable` or `ThumbnailPaintable`, then rendered by the `Picture` widget. Rotation is applied during rendering in `Picture::snapshot()` using GTK's snapshot API.

## Profiling

Set env vars before running to enable tracing backends:
- `IDENTITY_PROFILE_CHROME=1` — outputs `trace.json` (open in `chrome://tracing`)
- `IDENTITY_PROFILE_TRACY=1` — connects to Tracy profiler

Log level is controlled via `RUST_LOG` (default: INFO).
